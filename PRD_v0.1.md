# 相机翻译记忆库 PRD v0.1

更新时间：2026-07-02  
阶段：需求定义 / MVP 规划  
目标平台：手机 WebApp 优先，后续支持 iOS 原生外壳

## 1. 产品一句话

面向中国出境旅行用户的相机翻译记忆库：用户用自己的相机入口翻译路牌、菜单、包装、截图、书籍和工作资料时，系统自动识别、翻译、保存、拆分、分类，并把这些内容沉淀成可检索、可复习、可回顾的个人旅行语言记忆。

## 2. 目标用户

### 2.1 核心用户

- 中国人出境旅行者。
- 使用 iPhone。
- 经常通过相机、截图、网页、聊天截图理解外语信息。
- 不想额外花时间整理翻译记录。
- 希望旅行结束后留下可回顾、可搜索、可学习的语言记忆。

### 2.2 高频场景

| 场景 | 示例 | 用户意图 |
|---|---|---|
| 路牌 | 车站指示、禁令、景点标识 | 快速理解环境信息 |
| 菜单 | 餐厅菜单、饮品、过敏提示 | 点餐、记住常见表达 |
| 商品包装 | 成分、使用说明、价格标签 | 购物、避坑、复购 |
| 书籍文章 | 教材、展览介绍、说明文字 | 深度理解和学习 |
| 课堂材料 | 讲义、白板、资料页 | 课后复习 |
| 网页截图 | 酒店、交通、活动说明 | 行程决策 |
| 聊天截图 | 外语客服、朋友消息 | 沟通理解 |
| 工作资料 | 文档、邮件、会议材料 | 工作辅助和留档 |

## 3. 产品目标

### 3.1 用户目标

1. 拍照或上传后立即理解外语内容。
2. 系统自动保存所有翻译过的内容，不需要每次手动保存。
3. 自动拆分成单词、短语、句子、表达，并按重要性分权重。
4. 自动生成中文理解说明，帮助真正记住，而不是只给字面翻译。
5. 能按语言、时间、地点、场景、主题、来源图片搜索。
6. 旅行结束后能生成 QQ/微信可分享的旅行语言回忆册。

### 3.2 商业目标

| 层级 | 价值 |
|---|---|
| 免费本地版 | 建立使用习惯，完成拍照翻译入库和本地知识库 |
| 订阅云账户 | 云同步、云存储扩容、AI 深度整理、多设备访问、回忆册高级导出 |

## 4. 核心原则

1. 少打扰：用户主要动作是拍照、看翻译、继续旅行。
2. 自动化：保存、拆分、分类、去重、复习候选尽量自动完成。
3. 离线可用：没有网络时也能拍照识别、离线保存，联网后补齐翻译和 AI 整理。
4. 中国网络友好：联网能力不能依赖在中国大陆不稳定的服务。
5. 隐私优先：原图默认隐藏，隐私模糊可选，权限可选，数据本地免费可用。
6. 记忆优先：产品更像旅行语言经历库，不只是翻译工具或单词本。

## 6. 使用流程

### 6.1 新用户首次使用

1. 打开手机 WebApp。
2. 选择本地免费模式或登录云账户。
3. 选择默认保存策略：
   - 自动识别并保存全部内容。
   - 先显示预览，确认后入库。
4. 选择权限：
   - 相机：用于拍照翻译。
   - 相册：用于导入截图/照片。
   - 位置：默认可记录，按旅行城市自动归档。
   - 通知：用于复习提醒。
   - 文件存储：用于导出和本地备份。
   - 剪贴板：可选，用于从网页/聊天复制文本入库。
5. 选择离线语言包提示：
   - 告知用户可在 iPhone 系统中下载常用语言包。
   - 产品自身记录用户计划旅行地区并提示所需语言。

### 6.2 拍照翻译入库

1. 用户打开相机入口。
2. 拍摄路牌、菜单、包装、截图或文档。
3. 系统 OCR 识别图片文字。
4. 系统显示翻译结果：
   - 原文。
   - 自然中文翻译。
   - 重点推荐。
   - 可点选/划线/框选/长按重点内容。
5. 根据设置自动保存或预览确认后保存。
6. 系统自动生成记忆条目、拆分内容、分类、权重和复习卡片候选。

### 6.3 重点选择交互

| 交互 | 用途 |
|---|---|
| 手指划线 | 标记重点句子或短语 |
| 框选区域 | 选择菜单区域、说明书段落、路牌某块内容 |
| 点选文字块 | 快速选中 OCR 识别出的单词/短语/句子 |
| 长按一句话 | 查看详细解释并加入重点 |
| 自动推荐重点 | 系统根据陌生度、场景实用性、高频表达推荐 |
| 一键接受所有划线 | 用户认可系统建议后批量保存重点 |

## 7. 记忆内容结构

### 7.1 每条记忆需保存

| 字段 | 说明 |
|---|---|
| id | 唯一 ID |
| original_text | 原文 |
| translated_text | 中文自然译文 |
| literal_translation | 直译 |
| language | 识别语言 |
| confidence | OCR/语言识别置信度 |
| content_level | 单词 / 短语 / 句子 / 表达 / 段落 |
| weight | 重要性权重 |
| topic | 主题，如餐饮、交通、购物、住宿 |
| scene | 场景，如餐厅、车站、课堂、工作 |
| tone | 语气，如正式、礼貌、警告、口语 |
| usage | 使用场景解释 |
| example | 例句 |
| similar_expressions | 相似表达 |
| common_mistakes | 易错点 |
| source_image_id | 来源图片 ID |
| image_crop_id | 文字区域截图 ID |
| created_at | 拍摄/导入时间 |
| location_mode | GPS / 城市自动 / 手动 / 未记录 |
| location_text | 地点，如东京新宿某餐厅 |
| story | 场景故事，如 2026 年 7 月在东京餐厅菜单上看到 |
| review_status | 需要复习 / 已掌握 / 已忽略 |
| occurrence_count | 出现次数 |
| occurrence_ids | 每次出现的场景记录 |

### 7.2 同词/同表达去重策略

| 情况 | 处理方式 |
|---|---|
| 完全相同表达再次出现 | 合并为同一知识条目，出现次数 +1 |
| 同一表达不同场景 | 主条目合并，保留每次出现的独立场景 |
| 同一单词不同含义 | 按含义分子条目 |
| OCR 误差相近 | 提供可能重复提示，默认不强合并 |

## 8. 翻译解释标准

每条重点内容尽量包含：

| 模块 | 示例说明 |
|---|---|
| 一句中文解释 | 先用一句话说明用户真正需要理解的意思 |
| 直译 | 保留字面对照 |
| 自然译法 | 给出中文自然表达 |
| 使用场景 | 说明在哪些场景会看到/会用 |
| 语气 | 正式、礼貌、命令、警告、口语等 |
| 例句 | 生成短例句 |
| 相似表达 | 给出 1-3 个替代表达 |
| 易错点 | 提醒不要误解的地方 |

## 10. 离线与联网策略

### 10.1 离线必须支持

| 能力 | 说明 |
|---|---|
| 拍照 | 可调用 Web 相机能力 |
| 图片保存 | 原图和文字区域可本地保存 |
| OCR 队列 | 若本地 OCR 不可用，先保存图片和任务 |
| 基础元数据 | 时间、手动地点、用户选择内容 |
| 本地知识库 | 已有内容可搜索 |
| 待处理队列 | 有网后补翻译、解释、分类和同步 |

### 10.2 有网后补齐

| 任务 | 说明 |
|---|---|
| AI 翻译 | 生成自然中文解释 |
| 自动拆分 | 单词、短语、句子、表达 |
| 场景分类 | 餐饮、交通、购物、学习、工作等 |
| 去重合并 | 同词同表达合并，累计出现次数 |
| 云同步 | 云账户用户同步图片、文字、复习状态 |
| 回忆册生成 | 生成可分享内容 |

### 10.3 中国网络要求

联网能力应优先选择中国大陆稳定可访问的服务，避免第一版依赖在中国大陆不可稳定访问的第三方站点或 API。若使用海外模型，应通过合规、稳定的服务层封装。

## 11. 技术路线建议

### 11.1 第一阶段：PWA / 手机 WebApp

| 模块 | 技术方向 |
|---|---|
| 前端 | PWA，移动端优先 |
| 本地存储 | IndexedDB 保存文本、元数据、轻量图片索引 |
| 图片存储 | 本地 Blob / File System Access 能力视浏览器支持降级 |
| OCR | Web 端 OCR 或服务端 OCR；离线时进入待处理队列 |
| AI 整理 | 国内可访问 AI 服务或自建 API 层 |
| 导出 | CSV、Markdown、图片回忆册 |

### 11.2 第二阶段：iOS 原生外壳 + WebApp

纯 WebApp 很难直接调用 Apple Translation / VisionKit / 系统语言包。若要更好使用苹果本机语言包和相机文本能力，建议后续加一个很薄的 iOS 原生外壳：

| 原生层负责 | Web 层负责 |
|---|---|
| Apple Vision OCR | 知识库 |
| Apple Translation / 系统语言能力 | 复习 |
| 相机文本选择体验 | 搜索 |
| 本机权限和文件能力 | 账户和订阅 |
| 离线语言包检测提示 | 导出和回忆册 |

## 13. 隐私与安全

### 13.1 默认隐私体验

1. 原图可保存，但默认隐藏。
2. 文字区域截图可作为主要回顾素材。
3. 隐私模糊开关可选。
4. 用户可删除单条记忆、来源图片、某次旅行、全部本地数据。
5. 本地免费模式不强制登录。

## 19. 风险与待验证

| 风险 | 影响 | 处理 |
|---|---|---|
| 纯 WebApp 无法直接调用 Apple 系统语言包 | 影响离线翻译能力 | 第一版做离线保存和待处理；第二版加 iOS 外壳 |
| iOS Safari 对文件和本地存储限制 | 影响大量图片保存 | 控制图片大小，提供云订阅和导出备份 |
| Web OCR 速度和准确率不稳定 | 影响体验 | 支持服务端 OCR、有网后补处理 |
| 中国网络可用性 | 影响 AI 整理 | 选择国内稳定服务或自建 API 层 |
| 隐私图片误上传 | 影响信任 | 本地免费模式默认不上传；云同步前明确提示 |
| 自动分类不准 | 增加用户负担 | 支持用户一键修正，学习用户习惯 |
